SC26 Nixnan Tutorial: Floating-Point Precision Limits in Reaction-Diffusion CUDA

Format Legend:
FP16: 5-bit exp, 10-bit mantissa (fails earliest) BF16: 8-bit exp, 8-bit mantissa (range okay, precision compromised) FP32: 8-bit exp, 24-bit mantissa (reference precision)

1. Overflow Timeline

What: Step count at which each format first overflows to infinity.

Key finding: FP16 fails 6.3× earlier (step 300 vs 1900) due to limited exponent range (2¹⁵ vs 2¹²⁷).

Physics: Reaction term grows ~1.05× per step; each format climbs exponent axis until saturation.

2. Exception Propagation

What: Per-format count of NaN and Infinity exceptions (log scale).

Key finding: FP16 shows 100K+ NaN repeats vs 32K for BF16 — cascade of broken stencil operations.

Teaching moment: Infinity triggers NaN in next iteration (inf - 2·inf + inf = NaN).

3. Dynamic Range

What: Exponent range (min to max representable) for each format.

FP16 constraint: Stuck at [-14, 15] (29 bits total). Interior nodes reach exponent 15 by step ~227.

Hidden difference: BF16 and FP32 share identical exponent range but diverge in mantissa fidelity.

4. Per-Instruction Error Hotspots

Hottest instructions by format:

  • FP16: HADD2, HFMA2 (packed FP16 ops) — NaN/Inf propagation in stencil
  • BF16: HFMA2.BF16_V2 — FMA (fused multiply-add) is error hotspot
  • FP32: FADD, FFMA — floating-point addition/FMA lead NaN cascade

Stencil anatomy: Laplacian (u_{i+1} - 2u_i + u_{i-1}) dominates error frequency. Second-difference cancellation is the critical path.

5. Mantissa Precision Impact

Left: Mantissa width — BF16's 8 bits vs FP32's 24 bits.

Right: Diffusion term fidelity in PDE. BF16 (~2⁻⁸·u) is below noise floor → physics lost.

Hook: Same exponent range, same overflow step, wildly different precision for the gradient.

Interpretation Guide

  • FP16 subnormals: Expected zero in this run (high growth rate λ=50). Lower λ or raise dt to push boundary diffusion below 2⁻¹⁴ and observe subnormal underflow.
  • BF16 vs FP32 parity: Same overflow step (~1900), identical exponent histograms, but BF16's 8-bit mantissa renders diffusion invisible — precision loss not flagged by exceptions.
  • max_element() silent failure: After NaN propagates through interior, all comparisons with NaN return false → reports 0.0 (boundary), not inf/NaN. Classic silent failure mode.
  • Exception timeline: Infinity appears ~step 227 (FP16) or ~1818 (BF16/FP32); NaN follows in next stencil iteration due to inf - 2·inf + inf arithmetic.
  • Diffusion flushing: FP16 shows 22 Infinity + 22 NaN unique instances (100K repeats) vs BF16's 6 Infinity + 15 NaN. Repetition factor is cascade depth in interior.
